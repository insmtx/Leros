# 产品需求文档 (PRD) - Leros

> **最后更新：2026-05-21**
> **版本：3.0**


## 1. 产品定位

Leros 是一个面向企业与团队的 AI 工作协作系统（AI Workspace）。

系统通过以下核心要素构建可持续协作、可执行、可审计的 AI 工作环境：

- **项目（Project）** — 长期工作的核心容器
- **任务（Task）** — 最小执行单元
- **AI队友（Agent）** — 人格化执行者
- **技能（Skill）** — 可复用的能力模块
- **知识库（Knowledge）** — 长期记忆与上下文

Leros 不只是聊天工具。它更像：

- AI Native 的 Jira — 以任务驱动协作
- AI Native 的 Notion — 以知识组织工作
- AI Native 的 Cursor Team — 人机协同开发
- AI Native 的自动化工作系统 — 可持续执行的 AI 流水线

用户可以像组织真实团队一样：创建 AI 队友、分配任务、绑定技能、共享知识、跟踪项目、审计执行过程 — 让 AI 真正参与长期工作流，而不是一次性对话。


## 2. 产品核心概念

### 2.1 核心业务模型

```
任务（Task）  ← AI队友（Agent）执行  ← 绑定技能（Skill）
项目（Project）→ 组织多个任务和 AI队友
知识库（Knowledge）→ 为 AI队友提供上下文
工作台（Workspace）→ 统一工作入口
```

### 2.2 核心实体

| 实体 | 定义 | 职责 |
|------|------|------|
| 任务（Task） | 最小执行单元 | 定义要做什么，谁来做，输出什么 |
| 项目（Project） | 长期工作容器 | 组织任务、AI队友、上下文、记忆 |
| AI队友（Agent） | 人格化执行者 | 理解任务、调用技能、产出结果 |
| 技能（Skill） | 可复用能力模块 | 定义输入、输出、执行流程 |
| 知识库（Knowledge） | 持久化记忆 | 提供领域知识、项目上下文 |

### 2.3 协作关系

```
项目
 ├── 任务 ← AI队友执行 ← 技能
 ├── AI队友（多个，不同角色）
 ├── 知识库（项目专属）
 ├── 项目记忆（历史决策+任务结果）
 └── 长期上下文（目标+规范+业务资料）

平台全局
 ├── 全局技能库
 ├── 全局知识库
 └── AI队友运行时
```

### 2.4 与传统工具的对应

| Leros | 类比 |
|-------|------|
| 项目 | Jira Project + Notion Page |
| 任务 | Jira Issue + GitHub Issue |
| AI队友 | 团队成员（PM/Dev/QA） |
| 技能 | 工具 + API + 工作流 |
| 知识库 | Confluence + 代码文档 |


## 3. 产品信息架构（IA）

### 3.1 一级导航结构

```
工作台
任务
技能
知识库
项目
AI队友

```

### 3.2 导航定义

#### 工作台

用户的统一工作入口。

展示：
- 最近任务
- 待处理事项
- AI 协作动态
- 最近项目
- AI队友状态
- 快捷创建入口

#### 任务

任务是系统中的最小执行单元。

用户可以：
- 创建任务
- 指派 AI队友
- 跟踪执行状态
- 查看执行过程
- 接管任务
- 审核结果

#### 技能

技能是 AI 的可复用能力模块。

例如：PR审查、文档撰写、SQL分析、数据总结、Web搜索、CAD分析。

技能可以被：
- AI队友调用
- 任务直接调用
- 工作流复用

#### 知识库

知识库用于提供长期上下文与企业知识。

支持：文档知识、代码知识、项目知识、FAQ、向量检索、图谱检索。

#### 项目

项目用于组织长期任务、上下文和协作关系。

项目包含：项目目标、长期上下文、任务列表、AI队友、项目知识、项目记忆。

项目是长期工作的核心容器。

#### 个人中心

个人中心是用户管理个人配置和偏好的入口。

包含：
- **个人信息：** 头像、昵称、联系方式
- **账号安全：** 密码修改、登录记录、多因素认证
- **API 密钥：** 管理个人 API Token，用于外部系统集成
- **通知偏好：** 配置消息通知的渠道和频率
- **界面偏好：** 主题、语言、时区等个性化设置
- **操作日志：** 个人操作历史记录
- **使用统计：** 个人任务量、资源使用情况概览


## 4. 工作台

### 4.1 定位

工作台是用户进入系统后的统一工作入口，聚合展示用户关注的核心信息，提供快捷操作入口。

### 4.2 核心模块

**最近任务：** 展示当前用户最近创建或指派的待处理任务。

**待处理事项：** 需要用户审批、验收或接管的任务列表。

**AI协作动态：** 系统内 AI队友的执行动态，包括任务完成、审批请求、异常事件等。

**最近项目：** 用户最近参与的项目列表，支持快速跳转。

**AI队友状态：** 用户常用的 AI队友当前在线状态和忙碌情况。

**快捷创建入口：** 快速创建任务、项目或对话。

### 4.3 个性化

- 支持用户自定义工作台布局
- 支持 Pin 住常用项目和 AI队友
- 支持设置通知偏好


## 5. 任务系统

### 5.1 定位

任务是系统中的最小执行单元。任务驱动 AI队友工作，是人和 AI 协作的基本单位。

### 5.2 任务生命周期

```
创建 → 待分配 → 执行中 → 待审批 → 完成
                    → 失败
                    → 已取消
                    → 已暂停
```

**创建：** 用户创建任务，填写任务描述、目标、预期输出。

**待分配：** 任务等待分配 AI队友或人工执行。

**执行中：** AI队友正在处理任务，或人工正在执行。

**待审批：** 任务执行到需要人工审批的节点，暂停等待审批。

**完成：** 任务成功完成，结果已输出。

**失败：** 任务执行失败，可能由于错误、超时等原因。

**已取消：** 任务被用户取消。

**已暂停：** 任务被用户临时暂停，可恢复。

### 5.3 任务类型

**聊天任务：** 用户与 AI队友的实时对话，不需要持续上下文。

**执行任务：** AI队友执行一个明确的操作（如运行脚本、生成报告）。

**文档任务：** AI队友撰写或编辑文档。

**代码任务：** AI队友编写、审查或修改代码。

**调研任务：** AI队友搜集信息并汇总结论。

**自动化任务：** 由事件或定时器自动触发的任务。

**多Agent协作任务：** 多个 AI队友协同完成的任务。

### 5.4 任务属性

**优先级：** 任务的优先级，影响执行顺序（紧急/高/中/低）。

**截止时间：** 任务需要完成的时间要求。

**标签：** 用于分类和筛选任务的标签。

**关联项目：** 任务所属的项目。

**关联 AI队友：** 执行任务的 AI队友。

**子任务：** 一个任务可以拆分为多个子任务。

**依赖关系：** 任务之间的依赖关系。

**任务上下文：** 任务相关的背景信息、参考资料。

**预期输出：** 任务完成后的产出物描述。

### 5.5 人机协作

**AI执行：** 默认由 AI队友自动执行任务。

**用户接管：** 用户可以随时接管任务，转为人工执行。

**用户审批：** 关键节点需要用户审批后才能继续。

**用户补充上下文：** 用户可以在任务执行过程中补充信息。

**用户中断：** 用户可以中断正在执行的任务，修改指令后继续。

**用户恢复：** 中断或暂停的任务可以恢复执行。

### 5.6 定时任务

支持周期性任务配置（Cron 表达式），用于：
- 每日项目进展摘要
- 定期代码仓库巡检
- 定时知识库同步
- 定时数据分析
- 定时运营报告生成


## 6. 项目系统

### 6.1 定位

项目是系统中组织长期工作的一级核心实体。

项目承载：长期目标、上下文、协作关系、任务、AI队友、知识。

项目是 AI 长期记忆和持续协作的核心容器。

### 6.2 项目组成

```
项目
 ├── 项目信息（名称、描述、目标、状态）
 ├── 任务列表
 ├── AI队友（绑定多个 AI队友，不同角色）
 ├── 项目知识库（项目专属知识）
 ├── 项目记忆（系统自动沉淀）
 ├── 项目技能（项目私有技能）
 └── 项目成员（参与的人类用户）
```

### 6.3 项目能力

**项目上下文：** 维护长期背景信息，包括商业目标、技术架构、项目规范、业务资料。

**项目任务：** 项目下可创建多个任务，任务共享项目上下文。

**项目AI队友：** 项目可以绑定多个 AI队友，例如 PM、架构师、开发、测试、安全审计。

**项目知识库：** 项目可以拥有独立的知识库，存储项目相关的文档、代码、规范。

**项目记忆：** 系统自动沉淀历史决策、任务结果、讨论过程、执行经验。

**项目模板：** 支持将项目保存为模板，快速创建同类项目。

### 6.4 项目生命周期

```
创建 → 进行中 → 暂停 → 已完成 → 已归档
```

- **创建：** 填写项目信息，设定目标和范围
- **进行中：** 项目活跃，任务正在执行
- **暂停：** 项目暂时中止，可恢复
- **已完成：** 项目目标已达成
- **已归档：** 项目关闭，数据保留只读


## 7. AI队友系统

### 7.1 定位

AI队友是可长期协作的 AI 执行角色。AI队友不再是平台中心，而是任务执行角色。

### 7.2 AI队友核心组成

```
AI队友 = 角色 + 技能 + 记忆 + 执行环境
```

### 7.3 角色（Role）

**基础信息：** 名称、头像、描述。

**职责定义：** 负责的工作领域，例如"代码审查"、"问题解答"、"数据分析"。

**工作边界：** 不能做的事情，例如"不允许修改生产环境配置"、"不允许访问敏感数据"。

**系统提示词：** 行为准则和回复风格。

**输出格式：** 输出结果的格式要求。

**拒绝策略：** 超出职责范围或违反边界时的回应方式。

**审批场景：** 哪些操作需要人工审批后才能执行。

### 7.4 技能（Skill）

AI队友绑定的可执行能力模块。

**绑定方式：**
- 从全局技能库选取
- 从项目技能库选取
- 创建自定义技能

**技能权限：** 控制 AI队友对每个技能的使用权限（允许/禁止/需审批）。

### 7.5 记忆（Memory）

AI队友可访问的上下文。

**记忆来源：**
- 项目记忆（当前参与的项目）
- 知识库（已授权的知识库）
- 历史任务（之前执行过的任务）
- 长期偏好（用户设定的偏好）

### 7.6 执行环境（Runtime）

AI队友实际工作的运行环境。

**执行环境配置：**
- 默认模型
- 可用工具
- 工作节点
- 网络策略
- 资源限制

### 7.7 协作模式

**单人模式：** 一个 AI队友独立完成任务。

**主从模式：** 主 AI队友拆解任务，子 AI队友分别执行。

**审查模式：** 审查 AI队友检查执行 AI队友的结果。

**流水线模式：** 多个 AI队友形成执行流水线。

**并行模式：** 多个专业 AI队友从不同角度给出结论。


## 8. 技能系统

### 8.1 定位

技能是平台中的标准能力单元，是平台的核心资产。

技能定义：输入结构、输出结构、所需工具、所需模型、执行流程、权限需求。

### 8.2 技能分类

| 分类 | 说明 | 来源 |
|------|------|------|
| 系统技能 | 平台内置的标准能力 | 平台提供 |
| 项目技能 | 项目私有的能力 | 项目创建 |
| 团队技能 | 组织共享的能力 | 团队创建 |
| 市场技能 | 来自第三方的能力 | 技能市场 |

### 8.3 技能组成

**元信息：** 名称、描述、版本、分类、标签。

**输入Schema：** 技能调用的输入参数定义。

**输出Schema：** 技能输出的结果结构定义。

**执行流程：** 技能的内部执行逻辑（单步工作流或多步工作流）。

**所需工具：** 技能依赖的工具列表。

**所需模型：** 技能推荐或要求的 AI 模型。

**权限声明：** 技能执行所需的权限（文件读写、网络访问、外部API调用等）。

### 8.4 技能类型

**工具技能：** 直接调用外部工具或 API 的技能，如 Web搜索、文件操作。

**推理技能：** 基于 LLM 的推理能力，如代码审查、文本摘要、分类判断。

**工作流技能：** 多步骤编排的技能，如 PR审查工作流、Bug排查工作流。

**MCP服务：** 通过标准协议接入的外部服务能力。

### 8.5 技能管理

- 技能注册：注册新的技能
- 技能分类：按功能领域分类
- 版本管理：管理技能的版本和变更
- 依赖声明：声明技能的依赖关系
- 安全审核：对技能进行安全审核
- 启用和停用：控制技能的可用性
- 使用统计：统计技能的使用频率和效果


## 9. 知识库系统

### 9.1 定位

知识库为 AI队友提供企业内部上下文信息，是 AI 长期记忆的持久化存储。

### 9.2 知识类型

**文档知识：** 技术文档、产品文档、规范文档等。

**代码知识：** 代码仓库结构、API 定义、代码规范等。

**项目知识：** 项目目标、架构决策、业务资料等。

**FAQ：** 常见问题与答案。

**外部知识：** 通过连接器获取的外部知识源。

### 9.3 知识管理

**知识导入：** 从文档、代码仓库、数据库等来源导入知识。

**知识分类：** 按领域、项目、部门等维度分类。

**知识更新：** 支持增量更新和自动同步。

**权限控制：** 控制不同 AI队友和用户可访问的知识范围。

**使用统计：** 统计知识的使用情况。

### 9.4 知识检索

**全文检索：** 基于关键词的全文搜索。

**向量检索：** 基于语义的向量相似度搜索。

**混合检索：** 全文检索与向量检索结合。

**图谱检索：** 基于知识图谱的关系检索。

**RAG 集成：** 与 AI队友执行环境集成的检索增强生成。

### 9.5 知识库级别

| 级别 | 范围 | 说明 |
|------|------|------|
| 全局知识库 | 全平台 | 企业级通用知识 |
| 团队知识库 | 组织单元 | 团队共享知识 |
| 项目知识库 | 单个项目 | 项目专属知识 |
| AI队友知识库 | 单个AI队友 | 个人偏好和记忆 |


## 10. 平台引擎

### 10.1 定位

平台引擎为 AI队友提供真实的工作环境，包括 AI 模型、执行节点、外部连接和基础设施。

### 10.2 AI 模型管理

**模型供应商管理：** 支持多家模型供应商和本地部署模型。

**模型目录：** 维护模型名称、能力标签、上下文长度、成本、延迟、适用场景等信息。

**凭证管理：** 管理 API 密钥、组织 ID 等凭证信息，支持密钥轮换。

**路由策略：** 按 AI队友、任务类型、成本预算选择合适的模型。

**降级策略：** 主模型失败后切换备用模型。

**调用审计：** 记录模型调用、token 用量、耗时、错误和成本。

### 10.3 工作节点

工作节点是 AI队友执行真实任务的基础设施。

**支持的节点类型：**
- 容器沙箱节点：基于容器技术的隔离执行环境
- 远程执行节点：连接到远程服务器执行任务
- Browser 节点：浏览器自动化执行环境（规划中）
- Code Runtime：代码执行运行时（规划中）

**节点能力：**
- 节点健康检查
- 资源容量管理（CPU、内存、磁盘）
- 工作目录管理
- 资源限制
- 执行审计（命令日志、文件变更、执行产物）
- 沙箱生命周期管理

### 10.4 外部连接

外部连接解决"外部系统如何触发任务"的问题。

**支持的外部系统：**
- 代码平台：GitHub、GitLab、Gitee 等
- 事件通知：Webhook、API 回调等
- 业务系统：工单系统、审批系统、告警系统等

**典型通知类型：**
- PR/MR 创建、更新、评论
- Issue 创建、评论
- 监控告警触发/恢复
- 工单状态变更

**连接能力：**
- 事件类型配置
- 请求签名和密钥管理
- 内容映射为任务
- 幂等处理
- 失败重试

### 10.5 IM 身份

IM 身份用于将 AI队友绑定到企业即时通讯工具中的机器人。

**支持的 IM 渠道：** 企业微信、飞书、钉钉、Slack 等。

**典型能力：**
- 私聊消息
- 群聊消息
- @ 消息触发
- 卡片消息
- 文件消息
- 身份映射
- 消息审计


## 11. 企业级治理

### 11.1 多租户隔离

- 租户管理：支持多个企业租户
- 数据隔离：不同租户的数据完全隔离
- 资源隔离：不同租户的资源独立分配

### 11.2 用户和权限管理

- 用户管理：管理平台用户
- 组织管理：管理企业的组织和部门结构
- RBAC 权限：基于角色的权限控制
- 能力权限：控制谁能使用哪些技能和 AI队友

### 11.3 审计能力

## 12. MVP 规划

### 12.1 MVP 目标

第一版聚焦"AI 代码协作工作台"，实现项目驱动的 AI 协作核心闭环。

### 12.2 MVP 核心能力

**项目系统：**
- 创建和管理项目
- 项目上下文维护
- 项目任务管理

**任务系统：**
- 任务创建和执行
- 任务状态跟踪
- 任务执行记录
- 基本的人机协作（审批节点）

**AI队友系统：**
- 创建和配置 AI队友
- 角色配置（职责范围、工作边界）
- 技能绑定
- 模型配置

**技能系统：**
- 至少 1 个系统技能（PR审查）
- 技能绑定到 AI队友

**平台引擎：**
- AI 模型调用
- 容器沙箱执行环境
- GitHub 外部连接（支持 PR 事件触发任务）

**工作台：**
- 最近任务展示
- 快捷创建入口

### 12.3 MVP 端到端场景

1. 用户创建项目
2. 用户在项目中创建 PR审查 AI队友
3. 用户为 AI队友绑定 PR审查技能和模型
4. 开发者在 GitHub 创建 PR
5. GitHub Webhook 触发 AI队友
6. AI队友拉取代码变更，在工作节点中分析代码
7. AI队友生成审查结果，回写到 PR
8. 任务记录和执行轨迹保存到项目
9. 用户在工作台查看结果

### 12.4 MVP 验收标准

- 可以创建项目并在项目下管理任务
- 可以创建和配置 PR审查 AI队友
- 可以绑定模型、技能和容器工作节点
- 可以通过 GitHub PR 事件触发任务
- 可以在工作节点中执行代码分析
- 可以回写 PR 审查评论
- 可以查看任务执行记录
- 端到端延迟小于 30 秒
- 系统稳定性大于 99%
